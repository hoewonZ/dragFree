# dragFree 下一步计划大纲

## 目标

在 Windows 完成可运行 MVP 开发验证，并具备 macOS/Windows 打包能力（macOS 通过 CI/macOS 环境完成签名与公证链路）。

## 里程碑

### M1：项目骨架与基础运行

- 初始化 Electron 工程（Main/Renderer 结构、脚本、基础配置）。
- 建立托盘常驻与配置窗显示/隐藏流程。
- 引入配置存储（默认值：复制、1.5s 展开延时、成功静默）。
- 验证：Windows 启动后可托盘驻留，配置窗可打开/隐藏。

### M2：热区与拖拽触发链路

- 实现透明置顶 `OverlayWindow` 热区检测。
- 触发后从固定方向滑出目录面板。
- 实现取消区判定与普通拖拽回退。
- 验证：拖拽进入热区触发面板，离开有效区可取消并收起。

### M3：目录交互体验

- 展示常用目录与 `+` 入口。
- 悬停目录项立即放大反馈。
- 计时 `1.5s` 自动展开一级子目录，支持连续展开。
- 子目录超量滚动展示，父级始终可见。
- 验证：交互流畅，展开与滚动行为符合预期。

### M4：文件路由与冲突处理

- 支持文件 + 文件夹拖放目标目录。
- 执行复制/移动（默认复制，可配置切换）。
- 同名自动重命名规则落地（文件与文件夹）。
- 失败与取消通知，成功静默。
- 验证：批量拖放、冲突、权限异常、取消流程均可正确处理。

### M5：稳定性与打包发布

- 多屏与缩放适配、异常保护、日志补充。
- Windows 打包验收（安装、启动、托盘、拖拽主流程）。
- macOS CI 打包链路（签名、公证、staple）配置。
- 验证：生成可安装产物并通过核心冒烟测试。

## 执行顺序建议

1. 先完成 M1-M2，建立可观察的最小闭环。
2. 再做 M3，优先交互正确性。
3. 接着做 M4，保证文件操作可靠。
4. 最后做 M5，集中处理发布与兼容性。

## 每个里程碑的最小验收产出

- 可运行演示（录屏或步骤）
- 一组最小自动化测试（核心函数/状态机）
- 手动测试清单（本里程碑涉及场景）
- 变更记录（功能点与已知限制）
