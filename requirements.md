# dragFree 需求汇总

## 原始目标

1. 基于 Electron 开发桌面端应用，支持打包到 macOS 和 Windows，且可在 Windows 上完成开发验证。
2. 应用启动后显示小窗体做配置，配置完成后隐藏窗体并在后台运行（macOS 顶栏图标、Windows 托盘图标）。
3. 用户可配置常用目录；当用户把文件拖拽到桌面指定区域（如顶部、左上角等）时，弹出或滑出已配置目录图标，拖放后自动保存到对应目录。
4. 目录层级交互：悬停 1-2 秒自动展开一级子目录；拖拽滑过目录时有放大反馈；继续停留可继续展开。子目录过多时使用可滚动区域，展示固定数量（可配置）并始终保留父级可见。
5. 当拖拽滑动超出指定区域时，自动取消此功能，回退为普通拖拽行为。

## 已确认的产品决策

- 触发区形态：`屏幕边缘热区`（A）
- 保存策略：`可配置，默认复制`（C）
- 面板出现方式：`从热区固定方向滑出`（A）
- 技术实现：接受透明无边框置顶悬浮层用于接收拖拽（A）
- 首版支持对象：`文件 + 文件夹`（B）
- 文件夹处理策略：`可配置，默认复制`（C）
- 同名冲突策略：`自动重命名 (1)`（A）
- 首版热区数量：`1 个热区`（A）
- “+ 新建文件夹”命名：`每次弹输入框`（A）
