> **收录规则**
>
> - 按 Git 提交时间 **由旧到新** 排列（同一日期段内同样为旧 → 新）。
> - **不收录** 单独的 `chore(release): bump version …` 提交。
> - **不收录** 仅修改 `RELEASE_HISTORY.md`、`COMMIT_HISTORY.md`、`AGENTS.md`（可单独或组合出现）的维护性提交；这些提交仍保留在 Git 历史中，仅不出现在本文件。
> - 若某次提交在提升版本号的同时包含功能/修复改动（例如 `package.json` 与业务代码同批变更），**仍收录**，并在摘要中注明版本变化。
> - 每条摘要尽量 **一行**、只写增改了什么（与早期条目风格一致）；不写冗长分号链或实现细节，细节以提交说明与代码为准。
