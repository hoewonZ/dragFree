# dragFree：设计背景与应用概述（功能总览）

> 本文用于梳理当前仓库在 **2026-03-31** 之前已实现的主要功能点、关键修复与优化，并给出程序的设计背景与整体架构概述。

## 设计背景

Windows 桌面使用中，用户经常需要把一批文件快速投放到某些“常用目录”（例如项目目录、素材目录、下载归档目录）。传统方式要么频繁切换资源管理器窗口，要么依赖固定位置的窗口停靠；当屏幕多、窗口多时操作成本更高。

`dragFree` 的目标是把“拖拽投放”变成一个**低摩擦、高可预期**的流程：

- 鼠标拖动文件时进入屏幕边缘热区即可唤出面板，不必提前打开目标目录窗口。
- 面板展示常用目录，并支持悬停预览/展开子目录，减少层级点击。
- 投放过程提供轻量进度/结果反馈，不打断拖拽操作的连贯性。

## 应用概述（用户视角）

### 核心使用流程

- **热区触发**：拖动文件进入屏幕顶部热区（可配置），面板滑出。
- **目录选择**：在面板中选择常用文件夹或其子目录作为投放目标。
- **投放处理**：复制/移动（可切换），面板左下角显示处理进度与结果。
- **收尾**：结果提示停留一小段时间后，面板自动关闭；可选投放成功后打开目标文件夹。

### 主要窗口/界面

- **托盘（Tray）**：常驻入口（打开配置、退出）与部分状态提示。
- **Overlay 热区窗**：用于热区可视化与触发区域承载（透明/置顶）。
- **Panel 目录面板**：目录投放主界面（列表/平铺、面包屑、悬停展开等）。
- **配置页**：常用目录管理 + “配置（齿轮弹窗）”中展示/系统设置。

## 架构概述（代码结构）

### 主进程（`src/main/`）

- **`src/main/main.js`**
  - Electron 生命周期、窗口管理（Overlay/Panel/Config/Quick Open/New Folder）
  - IPC 协议入口（例如 `panel:drop-target`、`config:save` 等）
  - 拖拽会话与收尾编排（避免竞态、统一提示节奏）
  - 系统层能力：托盘菜单、开机自启动、日志写入等

- **`src/main/config-store.js`**
  - 默认配置与归一化（`DEFAULT_CONFIG`、`mergeConfig`）
  - 配置读写与安全兜底（数值夹取、枚举归一化）

- **`src/main/file-router.js`**
  - 文件投放路由（复制/移动、重名处理、错误收集、进度回调）
  - 跨盘移动（`EXDEV`）自动降级为复制+删除

### 渲染进程（`src/renderer/`）

- **`src/renderer/panel.html` / `panel-controller.js`**
  - 目录面板 UI 与拖拽命中判定
  - 进度/结果提示展示（左下角单一提示区）
  - “drop → dragend” 等事件时序下的会话防抖与收尾约束

- **`src/renderer/config.html`**
  - 常用目录管理
  - 齿轮弹窗：分区展示「显示设置 / 系统设置」

## 已实现功能点（按领域归类）

### 1) 拖拽投放与反馈

- **投放动作**：支持复制/移动（`behavior.defaultAction` + 拖拽中切换）。
- **投放进度**：投放处理时在 **面板左下角**显示“正在处理文件 x/y 文件名…”，不再弹出额外进度窗。
- **投放结果**：成功/部分失败/失败/取消均在左下角同一位置提示，并在可见时长后自动收尾关闭面板。
- **小文件不强制等待**：投放结果按实际完成时间立即显示，不再固定等待。
- **错误提示更直观**：失败使用更醒目的红色文字样式。

### 2) 常用目录与子目录交互

- **常用目录管理**：配置页添加/更改/删除/排序。
- **面包屑与列表/平铺**：面板支持面包屑导航与显示模式切换（具体实现以当前 UI 为准）。
- **悬停展开**：支持悬停触发子目录展示（含延时/冷却等参数）。

### 3) 系统设置

- **开机自启动（默认关闭）**
  - 配置项：`behavior.launchOnStartup`（默认 `false`）
  - 仅在 **打包环境**（`app.isPackaged`）下调用 `app.setLoginItemSettings` 生效，避免开发环境路径不稳定导致的启动项污染。

- **hotzone 调试日志开关（默认关闭）**
  - 配置项：`hotzone.hotzoneDebugLogEnabled`（默认 `false`）
  - 只有开启时才会写入 `hotzone-debug.log`，并在配置保存后即时同步启用/禁用。

### 4) 日志与可观测性

- **启动性能日志**：`startup-profile.log`（启动阶段埋点）
- **热区调试日志**：`hotzone-debug.log`（可配置开启）
- **拖拽失败日志**：`drag_fail_logs`（独立文件）
  - 记录：日期、拖拽模式（拖拽/快开）、失败文件名、错误码、错误码对应中文文案、原始原因、源路径。

## 关键 Bug 修复与优化（近期变更摘要）

以下为近期（2026-03-31）围绕投放反馈与系统设置的重点变更方向（按“问题→修复效果”总结）：

- **投放收尾竞态修复**
  - 问题：`drop` 后紧跟 `dragend`，可能抢先触发会话结束，导致结果提示闪烁或面板过早关闭。
  - 解决：主进程引入“路由进行中”标记，渲染进程对 `dragend`/手势收尾加防护，确保“先出结果→再收尾”。

- **进度反馈体验优化**
  - 问题：独立进度窗增加干扰；多屏时位置不符合用户当前屏幕上下文。
  - 解决：进度统一在面板左下角显示；移除独立进度窗。

- **小文件投放不延迟**
  - 问题：固定 baseline 等待导致小文件也要强行等满。
  - 解决：按实际处理完成时间出结果；仍保留结果提示可见时长避免一闪而过。

- **失败信息可理解性增强**
  - 增加错误码→中文提示映射，并在失败/部分失败提示里附带摘要；同时写入失败日志便于排查。

- **配置页信息架构调整**
  - 齿轮弹窗命名为「配置」，分为「显示设置」（上）与「系统设置」（下：开机自启动、hotzone 日志）。

## 配置项速览（与本文相关）

- **`behavior.launchOnStartup`**：开机自启动（默认关，仅打包后生效）
- **`hotzone.hotzoneDebugLogEnabled`**：hotzone-debug.log 写入开关（默认关）
- **`behavior.openTargetFolderOnDropSuccess`**：投放成功后打开目标文件夹（默认关）
